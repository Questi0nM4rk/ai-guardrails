# SPEC-006: Battle-Tested Defaults

## Status: Draft

---

## Overview

These defaults are the result of running ai-guardrails on real AI-maintained
repos and tuning until the false positive rate was acceptable without
sacrificing real enforcement. They are stored as static template files in
`src/templates/defaults/` and embedded into the binary at build time.

Profile `standard` uses these defaults. Profile `strict` removes additional
ignores. Profile `minimal` adds more.

## Python — ruff.toml defaults

Preserved from the battle-tested Python configuration:

```toml
# ai-guardrails:sha256=<computed>
# Generated by ai-guardrails — do not edit manually
# Regenerate: ai-guardrails generate

target-version = "py311"  # overridden by config.python_version
line-length = 88           # overridden by config.line_length
indent-width = 4           # overridden by config.indent_width
fix = false                # never auto-fix — force manual review
unsafe-fixes = false

[lint]
select = ["ALL"]

ignore = [
  # --- Formatter conflicts (ruff format handles these) ---
  # Always ignored — having ruff format AND these rules is contradictory.
  "W191",   # tab-indentation
  "E111",   # indentation-with-invalid-multiple
  "E114",   # indentation-with-invalid-multiple-comment
  "E117",   # over-indented
  "D206",   # indent-with-spaces
  "D300",   # triple-single-quotes
  "Q000",   # bad-quotes-inline-string
  "Q001",   # bad-quotes-multiline-string
  "Q002",   # bad-quotes-docstring
  "Q003",   # avoidable-escaped-quote
  "COM812", # missing-trailing-comma
  "COM819", # prohibited-trailing-comma
  "ISC001", # single-line-implicit-string-concatenation
  "ISC002", # multi-line-implicit-string-concatenation

  # --- Redundant with mypy --strict ---
  # ruff/ANN checks "is there a type hint?" — mypy validates correctness.
  # Running both generates duplicate noise with no additional safety.
  "ANN",

  # --- Docstrings — opt-in ---
  # Requiring docstrings on every function generates the most exceptions
  # and the most suppression comments. Off by default, opt-in via config.
  "D",

  # --- Development markers ---
  # Blocking commits over TODO/FIXME is counterproductive during active work.
  "FIX001", # line contains FIXME
  "FIX002", # line contains TODO
  "TD",     # flake8-todos

  # --- High false-positive / low value ---
  "ERA001", # commented-out-code (many false positives)
  "CPY001", # missing-copyright-notice
  "INP001", # implicit-namespace-package (missing __init__.py)
  "T201",   # print-found (legitimate in CLIs and scripts)
  "EM101",  # raw-string-in-exception
  "EM102",  # f-string-in-exception
  "TRY003", # raise-vanilla-args
  "S101",   # assert (legitimate for invariants)

[lint.per-file-ignores]
"tests/**/*.py" = [
  "ARG001",  # unused-function-argument (pytest fixtures)
  "ARG002",  # unused-method-argument (pytest fixtures)

"**/__init__.py" = [
  "F401",    # unused-import (re-exports)
]

[lint.pydocstyle]

[lint.flake8-type-checking]

runtime-evaluated-base-classes = ["pydantic.BaseModel"]

[lint.isort]
force-single-line = false
force-sort-within-sections = true
combine-as-imports = true
split-on-trailing-comma = true
force-to-top = ["__future__"]
required-imports = ["from __future__ import annotations"]

[lint.flake8-tidy-imports]
ban-relative-imports = "all"

[lint.flake8-tidy-imports.banned-api]
"typing.Optional".msg = "Use `X | None` instead"
"typing.Union".msg   = "Use `X | Y` instead"
"typing.List".msg    = "Use `list[X]` instead"

[lint.mccabe]

[lint.pylint]
```

## Shell — shellcheck defaults

No config file. Shellcheck runs with:

```
shellcheck --format=json1 --severity=warning <files>
```

Severity `warning` and above. Style findings (`SC2034` etc.) are excluded
unless profile is `strict`.

## Profile Differences

| Rule category | minimal | standard | strict |
|---|---|---|---|
| Formatter conflicts (ruff) | ignored | ignored | ignored |
| ANN (type hints) | ignored | ignored | **enforced** |
| D (docstrings) | ignored | ignored | ignored (too noisy) |
| T201 (print) | ignored | ignored | **enforced** |
| S101 (assert) | ignored | ignored | **enforced** |
| ERA001 (commented code) | ignored | ignored | **enforced** |
| mypy strict | off | on | on |
| shellcheck severity | error | warning | style |
| biome noExplicitAny | warn | error | error |

Profile is applied at config generation time — the generated files reflect the
profile. Changing profile requires `ai-guardrails generate` to rebuild configs.

## Template File Strategy

All defaults live in `src/templates/defaults/` as actual files.
Bun embeds them at build time via:

```typescript
// generators/ruff.ts
const RUFF_TEMPLATE = await Bun.file(
  new URL("../templates/defaults/ruff.toml", import.meta.url)
).text();
```

Variable substitution (line length, python version, etc.) is done with simple
string replacement on `{{line_length}}` tokens — no template engine needed.

This means the defaults are readable, editable, and reviewable as plain files
rather than being buried in TypeScript string literals.
