# SPEC-008: Per-Language Tools Reference

## Status: Draft
## Research date: March 2026

---

## Output format key

- **JSON** — single JSON document
- **NDJSON** — newline-delimited JSON (one object per line, NOT an array)
- **XML** — XML document
- **text** — human-readable text only
- **exit** — no machine output; use exit code

## Python

### ruff — lint + format (PRIMARY)

| Field | Value |
|-------|-------|
| Binary | `ruff` |
| Config file | `ruff.toml` (managed by ai-guardrails) |
| Lint command | `ruff check --output-format=json <dir>` |
| Format check | `ruff format --check <files>` |
| Output format | **JSON array** (lint); exit code (format) |
| Install check | `ruff --version` |
| Adoption | De facto Python standard since 2024 |

**JSON shape (lint):**
```json
[{ "code": "E501", "filename": "src/foo.py", "location": { "row": 10, "column": 5 }, "message": "..." }]
```

**Caveats:** `ruff format --check` has no JSON output — exit code only.

---

### pyright — type checking (PRIMARY, preferred over mypy)

| Field | Value |
|-------|-------|
| Binary | `pyright` (npm package) |
| Config file | `pyrightconfig.json` (managed by ai-guardrails) |
| Command | `pyright --outputjson <dir>` |
| Output format | **JSON** (single document, stable) |
| Install check | `pyright --version` |
| Adoption | Powers VS Code Pylance; widely adopted |

**JSON shape:**
```json
{
  "version": "1.1.x",
  "generalDiagnostics": [
    { "file": "src/foo.py", "range": { "start": { "line": 9, "character": 4 } },
      "severity": "error", "message": "...", "rule": "reportMissingTypeArgument" }
  ],
  "summary": { "filesAnalyzed": 12, "errorCount": 1, "warningCount": 0 }
}
```

**Why not mypy:** `mypy --output json` is explicitly unstable (long-standing open issues #10816, #13874). Format changes between minor versions. pyright's `--outputjson` is stable and single-document.

| Field | Value |
|-------|-------|

|-------|-------|
| Binary | `vulture` |
| Config file | `pyproject.toml [tool.vulture]` |

## TypeScript / JavaScript

### biome — lint + format + import sort (PRIMARY)

| Field | Value |
|-------|-------|
| Binary | `biome` |
| Config file | `biome.json` (managed by ai-guardrails) |
| Command | `biome ci --reporter=rdjson <dir>` |
| Output format | **JSON** (rdjson = Review Dog JSON, more stable than `json`) |
| SARIF variant | `biome ci --reporter=sarif <dir>` |
| Install check | `biome --version` |
| Adoption | Displacing ESLint + Prettier in new projects; v2 (2025) adds type-aware rules without tsc |

**Why rdjson over json:** The `--reporter=json` output is marked experimental and may change in patch releases. `rdjson` is more stable. `sarif` is also stable and preferred for CI upload.

**rdjson shape:**
```json
{
  "diagnostics": [
    { "location": { "path": { "text": "src/foo.ts" }, "range": { "start": { "line": 10, "column": 5 } } },
      "severity": "ERROR", "code": { "value": "lint/correctness/noUnusedVariables" }, "message": "..." }
  ]
}
```

### tsc — type checking (SECONDARY)

| Field | Value |
|-------|-------|
| Binary | `tsc` (from `typescript` npm package) |
| Config file | `tsconfig.json` (project-managed, not generated by ai-guardrails) |
| Command | `tsc --noEmit` |
| Output format | **text only** — no JSON output exists |
| Install check | `tsc --version` |

**Caveats:** biome v2 covers most type-aware lint rules without needing tsc. tsc is still the authoritative type-checker. Parse text output with regex: `file.ts(line,col): error TSxxxx: message`.

**Note on TypeScript 7 (Go rewrite):** `tsgo` is in preview as of Dec 2025. Not yet used in production. When stable, it replaces `tsc` with same flags.

| Output format | **JSON array** |
| Status | Production for non-type-aware rules; alpha for type-aware (Dec 2025) |

**Not included in MVP.** biome covers the same ground more completely. Add oxlint when type-aware rules stabilize (est. Q3 2026).

## Rust

### clippy — lint (PRIMARY)

| Field | Value |
|-------|-------|
| Binary | `cargo clippy` |
| Config file | `clippy.toml` (managed by ai-guardrails) |
| Command | `cargo clippy --message-format=json -- -D warnings` |
| Output format | **NDJSON** (one object per line — NOT an array) |
| Install check | `cargo clippy --version` |
| Adoption | Universal Rust standard |

**NDJSON parsing:** Read stdout line by line. Skip lines where `reason != "compiler-message"`. Extract from `.message.spans[]` for file/line/col.

**JSON shape (per line):**
```json
{ "reason": "compiler-message", "message": {
    "code": { "code": "clippy::needless_return" },
    "level": "warning",
    "spans": [{ "file_name": "src/main.rs", "line_start": 10, "column_start": 5, "is_primary": true }],
    "rendered": "..."
}}
```

## Go

### golangci-lint — meta-linter (PRIMARY, wraps go vet + staticcheck + more)

| Field | Value |
|-------|-------|
| Binary | `golangci-lint` |
| Config file | `.golangci.yml` (managed by ai-guardrails) |
| Command (v1.64+) | `golangci-lint run --output.json.path=stdout ./...` |
| Command (pre-v1.64) | `golangci-lint run --out-format=json ./...` |
| Output format | **JSON** |
| Install check | `golangci-lint --version` |
| Adoption | De facto Go CI standard (Kubernetes, Prometheus, Terraform) |

**CRITICAL CAVEAT:** The output flag changed in v1.64 (2025). Version-detect before choosing flag:
```typescript
const version = await getGolangciLintVersion(runner);
const jsonFlag = semver.gte(version, "1.64.0")
  ? "--output.json.path=stdout"
  : "--out-format=json";
```

**Why not separate go vet + staticcheck:** golangci-lint wraps both, plus 50+ more linters, in a single configurable run. No reason to invoke them separately when golangci-lint is available.

---

### govulncheck — security (SECONDARY)

| Field | Value |
|-------|-------|
| Binary | `govulncheck` |
| Config file | none (reads `go.mod`/`go.sum`) |
| Command | `govulncheck -json ./...` |
| Output format | **streaming NDJSON** |
| Exit code | **Always 0 with -json** — must parse output for findings |
| Install check | `govulncheck -version` |

**Caveats:** Exit code is 0 regardless of findings when using `-json`. Parse the stream for `{ "finding": ... }` events.

---

### Active runners for Go plugin

```
standard profile: golangci-lint
strict profile:   golangci-lint + govulncheck
minimal profile:  golangci-lint (minimal linter set in .golangci.yml)
```

## Shell

### shellcheck — lint (PRIMARY)

| Field | Value |
|-------|-------|
| Binary | `shellcheck` |
| Config file | `.shellcheckrc` (managed by ai-guardrails) |
| Command | `shellcheck --format=json1 <files>` |
| Output format | **JSON** (json1 preferred over json — includes more metadata) |
| File discovery | Glob: `**/*.{sh,bash,zsh,ksh}` |
| Install check | `shellcheck --version` |
| Adoption | Universal — 39k+ GitHub stars |

**json1 shape:**
```json
{ "comments": [
    { "file": "script.sh", "line": 10, "column": 5, "level": "error",
      "code": 2034, "message": "...", "fix": null }
]}
```

---

### shfmt — format (PRIMARY)

| Field | Value |
|-------|-------|
| Binary | `shfmt` |
| Config file | `.editorconfig` (reads indent settings) |
| Command | `shfmt -l <dir>` |
| Output format | **text** — lists filenames that differ; empty output = formatted |
| Install check | `shfmt --version` |

**Caveats:** `shfmt -l` exits 0 even if differences exist. Check if stdout is non-empty to detect violations.

## C / C++

### clang-tidy — lint (PRIMARY)

| Field | Value |
|-------|-------|
| Binary | `clang-tidy` |
| Config file | `.clang-tidy` (managed by ai-guardrails) |
| Command | `clang-tidy --quiet <files> 2>&1` |
| Output format | **text to stderr** — NO native JSON output |
| Requires | `compile_commands.json` for accurate analysis |
| Install check | `clang-tidy --version` |

**Text parsing pattern:**
```
src/foo.cpp:10:5: error: message [check-name]
```
Parse with regex: `/^(.+):(\d+):(\d+): (error|warning|note): (.+) \[(.+)\]$/`

**Alternative:** `clang-tidy-sarif` wrapper (Rust CLI tool) converts output to SARIF. Optional dependency.

## .NET / C\#

### dotnet build + Roslyn analyzers (PRIMARY)

| Field | Value |
|-------|-------|
| Binary | `dotnet` |
| Config files | `.editorconfig` + `Directory.Build.props` (both managed by ai-guardrails) |
| Command | `dotnet build --no-incremental -p:TreatWarningsAsErrors=true` |
| Output format | **text** — MSBuild format. No standalone JSON |
| TRX variant | `dotnet build --logger "trx;LogFileName=out.trx"` |
| Install check | `dotnet --version` |

**Key caveat:** Analyzers (Roslyn, SonarAnalyzer, Roslynator) run **at build time only**. There is no `dotnet analyze` command. The build IS the analysis. `TreatWarningsAsErrors=true` is required to fail on analyzer findings.

**Text parsing pattern:**
```
src\Foo.cs(10,5): error CA1234: message [Project.csproj]
```
Parse with regex: `/^(.+)\((\d+),(\d+)\): (error|warning) ([\w\d]+): (.+)$/`

---

### dotnet format — formatting (PRIMARY)

| Field | Value |
|-------|-------|
| Binary | `dotnet format` (built into .NET 6+ SDK) |
| Config file | `.editorconfig` |
| Command | `dotnet format --verify-no-changes --report out.json` |
| Output format | **JSON** (only with `--report`; exits **2** on violations, not 1) |
| Install check | `dotnet --version` |

**Exit code 2 (not 1)** for formatting violations — handle both `exitCode === 1` and `exitCode === 2` as failure.

---

### Active runners for .NET plugin

```
standard profile: dotnet-build + dotnet-format
strict profile:   dotnet-build + dotnet-format (TreatWarningsAsErrors is always on)
minimal profile:  dotnet-build only
```

## Output Format Summary

| Format type | Tools | Parsing |
|-------------|-------|---------|
| JSON array | ruff, bandit, biome (rdjson), shellcheck, oxlint, selene | `JSON.parse(stdout)` |
| JSON object | pyright, cargo-audit, govulncheck events | `JSON.parse(stdout)` |
| NDJSON | clippy, staticcheck, govulncheck | `stdout.split('\n').filter(Boolean).map(JSON.parse)` |
| Text (regex) | clang-tidy, cppcheck, dotnet build, codespell, markdownlint, tsc | Per-tool regex |
| Exit code | rustfmt, shfmt, clang-format, stylua, `ruff format` | `result.exitCode !== 0` |
| XML | cppcheck (stderr), dotnet trx | XML parser |
